## 关于 Gridea Pro

你可能听说过 [Gridea](https://open.gridea.dev/) —— 一款曾经帮助超过 28,000 位用户搭建独立博客的开源写作客户端。它在 GitHub 上收获了 10,000+ Stars，证明了一件事：**很多人想要自己的博客，只是被技术门槛挡在了门外。**

但旧版 Gridea 基于 Electron + Vue 2 构建，已经停更近 4 年。底层的同步机制不够稳定，主题生态停滞，技术栈也跟不上时代了。

**Gridea Pro 是对 Gridea 的全面重生。**

我们从零重写了整个项目。不是修修补补，而是带着旧版积累的所有经验教训，重新思考「一个博客工具应该是什么样子」——更快、更稳定、不依赖任何外部环境，下载即用。

如果你是 Gridea 老用户 —— 欢迎回来，你的数据可以无缝迁移过来。
如果你是新朋友 —— 来对地方了。

## 为什么要写博客

在动笔之前，也许你会犹豫：2026 年了，还有人看博客吗？公众号、小红书、知乎不是更方便？

这么说吧。

你在平台上写的内容，**不完全是你的**。它可能因为某个关键词被限流，可能因为某次政策调整被删除，可能因为平台衰落而随之消失。你精心写的万字长文，被算法折叠成信息流里一条滚动的卡片。阅读量取决于发布时间，而不是内容质量。

更根本的问题是——平台的利益和你的利益并不总是一致的。平台要的是留住用户、卖广告、做商业化。你要的是被阅读、被记住、被找到。

独立博客是你在互联网上真正拥有的一小块地方。**你的域名、你的设计、你的内容、你的数据——全部由你说了算。**

而且，恰恰是在 AI 生成内容泛滥的今天，一个有真实人格、有独立思考的博客，比以往任何时候都更珍贵。当所有人都在用 AI 批量生产「还不错」的内容时，读者反而更渴望看到一个真实的人在认真地写。

你不需要是作家。记录一次踩坑经历、分享一个学到的东西、写下某天的所思所想——这些不起眼的文字，积累起来就是独一无二的你。两年后回头看，你会庆幸自己写了下来。

**博客不是给别人看的，是给未来的自己留的。** 别人能看到，那是额外的收获。

## Gridea Pro 为你准备了什么

过去，搭建一个静态博客意味着：安装 Node.js 或 Go 环境，在终端敲一串命令，编辑 YAML 配置文件，配置 Git SSH 密钥，处理部署报错……每一步都可能劝退一个想写东西的人。

Gridea Pro 的目标很直接：**让写博客这件事，回到「写」本身。**

- **零门槛** — 不需要装 Git、不需要开终端、不需要碰配置文件。你面前这个客户端就是全部。
- **好用的编辑器** — 内置 Monaco 编辑器（和 VS Code 同款内核），完整的 Markdown 语法支持，脚注、任务列表、目录生成、Emoji 一应俱全。
- **一键发布** — 写完点「发布」，博客自动推送到 GitHub Pages、Gitee Pages 或 Vercel。底层用纯 Go 实现了完整的 Git 引擎，你的电脑上不需要安装任何额外环境。
- **主题随心换** — 内置多款精心设计的主题，点击即可切换，每个主题都支持可视化配置。不用改代码，不用碰模板文件。
- **Memo 速记** — 不是所有想法都值得写成一篇文章。打开「速记」，用一两句话记下灵感、碎片、感悟。热力图会帮你追踪创作频率，让坚持变得可视化。
- **评论互动** — 内置 Gitalk、Giscus、Waline、Twikoo 等 7 种主流评论系统的集成，让读者能找到你、和你对话。
- **SEO 友好** — 自动生成 Sitemap、RSS 订阅、结构化数据，让搜索引擎能发现你的内容。

## 你的数据，只属于你

这一点我们想特别强调。

你在 Gridea Pro 里写的每一篇文章、每一条速记、每一项配置，**全部以 Markdown 和 JSON 文件的形式保存在你本地的文件夹里**。没有云端数据库，没有账号系统，没有任何形式的锁定。

你可以用 VS Code 打开同一个目录编辑。你可以用 iCloud、OneDrive、坚果云同步到多台设备。你可以随时把整个文件夹打包备份。即使有一天你不再使用 Gridea Pro，这些 Markdown 文件依然可以被 Hugo、Hexo、Astro 或任何工具读取——因为它们就是通用的纯文本。

**你的文字不应该被任何工具绑架。**

## 现在，开始写点什么吧

这篇文章是你博客里的第一篇内容，你可以随时编辑或删除它。

但在删除之前，不妨先试试：点击左上角的「新建文章」，随便写点什么。不需要想标题，不需要纠结排版，就写下你现在脑海里的一个想法。哪怕只有一句话。

**最好的博客，是从第一篇开始的。**
